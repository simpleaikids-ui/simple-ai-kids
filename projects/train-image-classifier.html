Train an Image Classifier

Teach a neural network to tell apart things you care about — Pokémon, sneakers, your art. Use transfer learning so training takes 5 minutes, not 5 days.

What You'll Make

A working image classifier for 3–5 classes of your choosing. You'll collect a tiny dataset (50–100 images), fine-tune a pre-trained MobileNet, and test it on new images.

What You'll Need

  • Google Colab (free)
  • 50–100 images per class (you can download from the web)
  • Basic Python
  • About 2 hours

What You'll Learn

  • What transfer learning is and why it's the right call for small datasets
  • How to split data into train/validation sets
  • How to read a confusion matrix
🧑‍🍳 Grown-Up Notes Web scraping for training images is fine for personal projects but not for commercial use. Discuss image licensing briefly. Colab's free GPU is enough for this — no cost.

Steps

  1. Decide on 3 classes you care about — e.g. starter pokémon: bulbasaur, charmander, squirtle.
  2. Gather 50 images per class. Use bulk image downloader Chrome extensions from a search engine, or the bing-image-downloader Python package in Colab.
  3. Organize them into folders: data/train/bulbasaur/, etc.
  4. In Colab, install and import: 
    !pip install -q tensorflow
import tensorflow as tf
from tensorflow.keras import layers, Model
  5. Load data with an 80/20 split: 
    train = tf.keras.utils.image_dataset_from_directory("data", validation_split=0.2, subset="training", seed=42, image_size=(160,160), batch_size=32)
val   = tf.keras.utils.image_dataset_from_directory("data", validation_split=0.2, subset="validation", seed=42, image_size=(160,160), batch_size=32)
  6. Build the model — pre-trained MobileNetV2 + a new classifier head: 
    base = tf.keras.applications.MobileNetV2(input_shape=(160,160,3), include_top=False, weights="imagenet")
base.trainable = False
x = tf.keras.layers.GlobalAveragePooling2D()(base.output)
x = tf.keras.layers.Dense(len(train.class_names), activation="softmax")(x)
model = tf.keras.Model(base.input, x)
model.compile(optimizer="adam", loss="sparse_categorical_crossentropy", metrics=["accuracy"])
  7. Train for 10 epochs: model.fit(train, validation_data=val, epochs=10). Watch validation accuracy.
  8. Test on 3 new images the model hasn't seen. Plot the predictions.
  9. Print a confusion matrix to see which classes the model mixes up.
🪄 If something goes wrong… Accuracy stuck at 33% (random)? Your dataset is probably imbalanced or too small — get more images. Training accuracy 99% but validation 40%? Classic overfitting. Add data augmentation or more examples.

Try This Next

  • Fine-tune the base. Unfreeze the last few layers of MobileNet and train with a tiny learning rate. Usually +5–10% accuracy.
  • Grad-CAM. Visualize which parts of the image the model is using to decide — are they meaningful?
  • Deploy with Gradio. Wrap the model in a Gradio UI and share a live demo link.
