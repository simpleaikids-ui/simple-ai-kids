Tiny Transformer From Scratch

Follow Karpathy's "Let's build GPT" and implement a working transformer in PyTorch — attention, positional encoding, the works. Train it on Shakespeare.

What You'll Make

A ~200-line nanoGPT-style character-level transformer trained on the complete works of Shakespeare. You'll generate text samples and be able to explain every line.

What You'll Need

  • Colab or a local GPU
  • PyTorch and patience to work through details
  • Weekend

What You'll Learn

  • What attention actually computes (queries, keys, values)
  • Why positional encoding exists
  • How the training signal propagates through a decoder stack
🧑‍🍳 Grown-Up Notes Based on Andrej Karpathy's Zero-to-Hero series — the canonical intro. Watching the video first is strongly recommended (~2 hours). No API cost.

Steps

  1. Watch Karpathy's "Let's build GPT: from scratch, in code, spelled out." Takes 2 hours. Worth every minute.
  2. Download the Shakespeare dataset: 
    !wget https://raw.githubusercontent.com/karpathy/char-rnn/master/data/tinyshakespeare/input.txt
  3. Tokenize at character level. Build stoi (string→int) and itos maps. Encode the whole file as a tensor.
  4. Implement the single-head self-attention block from scratch: 
    class Head(nn.Module):
    def __init__(self, C, H):
        super().__init__()
        self.key   = nn.Linear(C, H, bias=False)
        self.query = nn.Linear(C, H, bias=False)
        self.value = nn.Linear(C, H, bias=False)
        self.register_buffer("tril", torch.tril(torch.ones(block_size, block_size)))
    def forward(self, x):
        B,T,C = x.shape
        k = self.key(x); q = self.query(x)
        wei = q @ k.transpose(-2,-1) * k.shape[-1]**-0.5
        wei = wei.masked_fill(self.tril[:T,:T] == 0, float("-inf"))
        wei = F.softmax(wei, dim=-1)
        return wei @ self.value(x)
  5. Extend to multi-head. Stack N blocks. Add layer norm and a small feed-forward.
  6. Write the training loop. AdamW, lr=3e-4, batch size 64, block_size 256. Expect 30 min on a T4.
  7. Target loss: ≤ 1.5. Above that, your model hasn't really learned yet.
  8. Generate samples: 
    ctx = torch.zeros((1,1), dtype=torch.long, device=device)
print(decode(model.generate(ctx, 500)[0].tolist()))
    It should produce fake-Shakespeare that looks remarkably plausible.
🪄 If something goes wrong… Loss stuck at ~3.5? Your embeddings, positional encoding, or mask shape is off. Print intermediate tensor shapes — print(x.shape) at every step until one surprises you.

Try This Next

  • Switch datasets. Train on your messaging history or a favorite book. See how voice transfers.
  • Byte-pair tokenization. Replace character-level with a tiny BPE tokenizer. Sample quality jumps.
  • Flash attention. Swap manual attention for PyTorch 2.0's scaled_dot_product_attention. 2–4× faster.
