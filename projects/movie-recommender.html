Movie Recommender

Build a "people who liked this also liked…" recommender using real data from MovieLens. Your model will learn what movies go together — and start guessing.

What You'll Make

A working movie recommender. Type in a movie you love, and it returns 5 other movies that similar viewers rated highly. No fancy UI yet — just a Colab notebook that prints real recommendations.

What You'll Need

  • A Google account (to use Google Colab)
  • About 2 hours
  • No installs — everything runs in the browser
  • Basic Python knowledge (variables, lists, functions)

What You'll Learn

  • What collaborative filtering is, and why Netflix & Spotify use it
  • How to work with a real dataset (loading, cleaning, filtering)
  • How to compute similarity between items using cosine similarity
🧑‍🍳 Grown-Up Notes Google Colab is free and runs in the browser. The MovieLens 100K dataset is a publicly available research dataset with no personal data — just anonymized user ratings. All computation happens on Google's servers; nothing is installed locally. Students should use a personal or school Google account; no credit card is required.

The Idea Behind It

If two people both rated Toy Story 5 stars, they probably have similar taste. If one of them also loved Finding Nemo, the other probably will too. Collaborative filtering turns this common-sense idea into math: for each movie, we build a "fingerprint" of how people rated it, then find movies with similar fingerprints.

Collaborative Filtering — Users × Movies → Similarity

Steps

  1. Open colab.research.google.com. Click File → New notebook. Sign in with your Google account.
  2. In the first cell, paste this code to download the dataset: 
    !wget -q https://files.grouplens.org/datasets/movielens/ml-latest-small.zip
!unzip -q -o ml-latest-small.zip
    Click the ▶ play button on the left of the cell. Wait a few seconds.
  3. Add a new cell and load the data: 
    import pandas as pd
ratings = pd.read_csv("ml-latest-small/ratings.csv")
movies  = pd.read_csv("ml-latest-small/movies.csv")
print(ratings.head())
print(movies.head())
    You should see tables of user ratings and movie titles.
  4. Pivot the ratings into a matrix of users × movies: 
    ratings_matrix = ratings.pivot_table(
    index="userId", columns="movieId", values="rating"
).fillna(0)
print(ratings_matrix.shape)
  5. Compute similarity between every pair of movies: 
    from sklearn.metrics.pairwise import cosine_similarity
movie_similarity = cosine_similarity(ratings_matrix.T)
print("Similarity matrix shape:", movie_similarity.shape)
  6. Write a helper function that takes a movie title and returns recommendations: 
    def recommend(title, n=5):
    match = movies[movies["title"].str.contains(title, case=False)]
    if match.empty:
        return f"No movie found containing '{title}'."
    movie_id = match.iloc[0]["movieId"]
    idx = ratings_matrix.columns.get_loc(movie_id)
    scores = list(enumerate(movie_similarity[idx]))
    scores = sorted(scores, key=lambda x: x[1], reverse=True)[1:n+1]
    rec_ids = [ratings_matrix.columns[i] for i, _ in scores]
    return movies[movies["movieId"].isin(rec_ids)]["title"].tolist()

print(recommend("Toy Story"))
  7. Try it with your favorite movies! Some will work, some won't (the dataset is small). See how it does with Star Wars, Harry Potter, Shrek.
🪄 If something goes wrong… Getting KeyError? The movie probably isn't in this dataset — try another. Weird recommendations? That's real! Small datasets give noisy results. In Try This Next, you'll explore why.

Try This Next

  • Upgrade the dataset. Swap in MovieLens 1M (10× more ratings). See if recommendations feel better.
  • Add a filter. Only recommend movies with at least 20 ratings — this removes noise from rarely-rated obscure titles.
  • Compare approaches. Try content-based filtering too (recommend based on genre tags). Which feels more useful?
  • Build a UI. Use Streamlit or Gradio to turn this into a real web app you can share.
