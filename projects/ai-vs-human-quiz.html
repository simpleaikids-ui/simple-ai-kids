AI vs. Human Quiz

Build a web page that shows mystery texts or images — half made by AI, half by humans. Players guess which is which, and you track the scores.

What You'll Make

A standalone static web page with 10 rounds. Each round shows a piece of content; the player picks AI or human. At the end, accuracy + which rounds stumped the most people.

What You'll Need

  • A text editor (VS Code, Sublime, or just the browser)
  • A collection of 20 pairs of AI/human content (you'll gather these)
  • Basic HTML + JavaScript
  • About 3 hours

What You'll Learn

  • How to build a small interactive web app with zero frameworks
  • How to measure human performance on a task
  • Which content types are easiest/hardest to tell apart
🧑‍🍳 Grown-Up Notes This project is also great social/ethics content. The teen has to collect both AI and human content — which means they'll learn what both look like up close. No backend needed, all client-side.

Steps

  1. Gather 10 human-made texts (from classic books, Wikipedia, news articles — cite each) and 10 AI-made texts (paste prompts into a chatbot, save outputs).
  2. Keep pairs of similar topics. AI haiku vs human haiku. AI product description vs human review.
  3. Create index.html with a minimal skeleton: 
    <!doctype html>
<html><head><title>AI or Human?</title></head>
<body>
  <div id="card"></div>
  <button onclick="guess('ai')">AI</button>
  <button onclick="guess('human')">Human</button>
  <div id="score"></div>
  <script src="quiz.js"></script>
</body></html>
  4. Create quiz.js with your data: 
    const rounds = [
  { text: "A soft rain fell on the windowsill…", source: "ai" },
  { text: "To be, or not to be…", source: "human" },
  // … 18 more
];
rounds.sort(() => Math.random() - 0.5);
let i = 0, right = 0;
function show() {
  if (i >= rounds.length) return finish();
  document.getElementById("card").textContent = rounds[i].text;
}
function guess(g) {
  if (g === rounds[i].source) right++;
  i++; show();
}
function finish() {
  document.getElementById("score").textContent = \`You got \${right}/\${rounds.length}\`;
}
show();
  5. Style it with CSS — a big text card, clear buttons.
  6. Test yourself first. Which ones tripped you up?
  7. Playtest with 5 friends. Record the accuracy for each round.
  8. Find the hardest round. Why was that one the trickiest?
🪄 If something goes wrong… Everyone gets 100%? Your examples are too obvious — the AI ones are typical GPT-ish. Use a better prompt: "write in the style of [specific author], 1 paragraph." Everyone gets 50% (random)? Great — that means AI is now that good!
