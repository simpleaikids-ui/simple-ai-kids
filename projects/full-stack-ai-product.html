Full-Stack AI Product

Build and deploy a real AI web app — from prompt design through UI, cost modeling, and latency tuning. Ship it to real users.

What You'll Make

A deployed AI web app — your choice of use case — with a real UI (React or Next.js), an LLM backend, cost tracking, and at least 5 real users testing it.

What You'll Need

  • Node.js, Next.js or React + a minimal Python backend (or Next.js API routes)
  • An Anthropic or OpenAI API key (\$5–20 of credit)
  • Vercel or Fly.io account (free tier)
  • About 2 weeks

What You'll Learn

  • How to design prompts that don't break in production
  • How to cost-model an LLM feature before shipping
  • How to collect real user feedback and act on it
🧑‍🍳 Grown-Up Notes Budget ~\$20 total. Watch the API spend dashboard daily. Discuss data-handling: never log personal info; never send private data to third-party models without disclosure.

Steps

  1. Pick a use case. Something specific: a tutor for one class, a tone-rewriter, a recipe generator from ingredients you have. Avoid "general chatbot."
  2. Design the prompt. Start on paper. Define the role, constraints, edge cases. Test in a playground with 10 real inputs.
  3. Cost model. Estimate tokens per request × price × expected daily requests = monthly cost. If this is >\$50, redesign.
  4. Scaffold a Next.js app: 
    npx create-next-app@latest my-ai-app --ts --tailwind --app
cd my-ai-app
npm i @anthropic-ai/sdk
  5. Create an API route app/api/generate/route.ts: 
    import Anthropic from "@anthropic-ai/sdk";
const client = new Anthropic();
export async function POST(req: Request) {
  const { input } = await req.json();
  const msg = await client.messages.create({
    model: "claude-haiku-4-5-20251001",
    max_tokens: 400,
    system: "You are a polite tone-rewriter. Rewrite the input keeping meaning but making it warmer.",
    messages: [{ role: "user", content: input }],
  });
  return Response.json({ text: msg.content[0].text });
}
  6. Build the UI (app/page.tsx) with a textarea, submit button, and result pane. Add loading state and an error state.
  7. Deploy. Push to GitHub, connect to Vercel. Add ANTHROPIC_API_KEY in Vercel env. Done.
  8. User test. Send to 5 friends. Record what breaks. Fix the top 2 issues.
  9. Write a retrospective. Cost per user. Latency p50/p95. Worst failure mode. What you'd change next time.
🪄 If something goes wrong… Rate-limit errors under load? Add exponential backoff. Unexpected API cost spike? Add per-user daily caps. Response feels wrong? 80% of the time it's the prompt, 20% the model.

Try This Next

  • Streaming. Stream tokens back to the browser so the UI fills in as the model types.
  • Cache. Cache recent responses for identical inputs. Cuts cost and latency.
  • A/B test prompts. Send 50/50 traffic to two prompts, measure thumbs up/down.
