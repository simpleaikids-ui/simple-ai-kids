Fine-Tune a Small LLM

Fine-tune Llama 3.2 1B on your own writing (or a favorite author's). Discover how much data it actually takes to change a model's voice.

What You'll Make

A LoRA-fine-tuned version of a 1B-parameter LLM that sounds like a target author or like you. You'll produce before/after samples and a short writeup.

What You'll Need

  • Colab Pro (\$10/mo, for A100) — or patience on T4
  • A Hugging Face account + Llama license accepted
  • 5–50 KB of target text (blog posts, essays, novel excerpts)
  • A weekend

What You'll Learn

  • What LoRA is and why it's the right tool for this
  • How to prepare a chat-format dataset
  • How tiny data changes outputs — and where it fails
🧑‍🍳 Grown-Up Notes Llama 3.2 is free but requires license acceptance on Hugging Face. LoRA adapters are only ~20 MB. No API cost. If using a dead author's work, double-check public domain status (pre-1928 in the US is safe).

Steps

  1. Gather your text. ~20 KB of your own writing is enough; ~200 KB gives noticeably more faithful voice.
  2. Format as a simple chat dataset — pairs of (prompt, response). Or plain completion lines — you pick the format.
  3. Accept the Llama 3.2 license on huggingface.co/meta-llama/Llama-3.2-1B.
  4. In Colab with GPU: 
    !pip install -q transformers peft trl datasets bitsandbytes
from transformers import AutoTokenizer, AutoModelForCausalLM
from peft import LoraConfig, get_peft_model
import torch

tok = AutoTokenizer.from_pretrained("meta-llama/Llama-3.2-1B")
model = AutoModelForCausalLM.from_pretrained("meta-llama/Llama-3.2-1B", torch_dtype=torch.bfloat16, device_map="auto")
  5. Apply LoRA: 
    lora = LoraConfig(r=16, lora_alpha=32, target_modules=["q_proj","v_proj"], lora_dropout=0.05, task_type="CAUSAL_LM")
model = get_peft_model(model, lora)
model.print_trainable_parameters()   # should be <1% of total
  6. Use trl.SFTTrainer with 3–5 epochs and batch_size=1 on T4, bs=4 on A100. Expect 30 min – 2 hours.
  7. Generate before/after samples for the same prompt. Save both. Show a friend.
  8. Write the reflection: where does the fine-tune pick up the voice? Where does it fail? What broke?
🪄 If something goes wrong… CUDA OOM on T4? Drop to r=8, bs=1, sequence length 512. Loss plateau after epoch 1? Your dataset might be too small or too repetitive. Garbled output? Your tokenizer and formatting are mismatched — print one batch and check.
