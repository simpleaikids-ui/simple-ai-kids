RAG-Powered Study Buddy

Build a chatbot that answers questions about your class notes. Learn the same Retrieval-Augmented Generation architecture real companies deploy — embeddings, vector search, and prompted LLM responses.

What You'll Make

A CLI (or optionally a small web UI) that takes a folder of your notes — PDFs, Markdown, or plain text — and lets you ask questions. The bot retrieves the most relevant chunks from your notes and uses an LLM to write a grounded answer with citations. This is the same architecture behind Notion Q&A, ChatGPT with custom data, and countless "chat with your docs" products.

What You'll Need

  • Python 3.10+ and a terminal
  • An Anthropic or OpenAI API key (~$1 of credit is enough for this project)
  • A folder of study notes (start with 5–20 pages)
  • About a weekend
  • Basic familiarity with Python, virtual environments, and pip

What You'll Learn

  • What embeddings are and how semantic similarity works
  • How vector databases enable fast retrieval across millions of chunks
  • Why chunking strategy matters (and how to break your notes correctly)
  • Prompt design for grounded, citation-backed answers
  • Real-world tradeoffs: cost per query, latency, and when RAG fails
🧑‍🍳 Grown-Up / Self Notes Keep the API key in a local .env file; never commit it. Watch your API usage — chatting about one document costs fractions of a cent, but embedding a 500-page book can run a dollar or two. If you're under 18, you'll need parent approval to create an API account.

RAG Pipeline — Index, Retrieve, Generate

The Architecture

RAG works in three phases:

  1. Index time: Split documents into chunks, embed each chunk into a vector, store in a database.
  2. Query time: Embed the user's question, find the most similar chunks in the database.
  3. Generation: Stuff the retrieved chunks into the LLM's context and ask for an answer.

Steps

  1. Set up your project: 
    mkdir study-buddy && cd study-buddy
python -m venv .venv
source .venv/bin/activate        # on Windows: .venv\Scripts\activate
pip install anthropic chromadb pypdf python-dotenv
    Create a .env file with ANTHROPIC_API_KEY=sk-ant-....
  2. Create a notes/ folder and drop some PDFs or .md files in it.
  3. Write ingest.py — load docs, chunk them, embed them, store them: 
    import os, glob
from pypdf import PdfReader
import chromadb
from dotenv import load_dotenv
load_dotenv()

client = chromadb.PersistentClient(path="./db")
coll = client.get_or_create_collection("notes")

def chunks(text, size=800, overlap=100):
    for i in range(0, len(text), size - overlap):
        yield text[i:i+size]

def load(path):
    if path.endswith(".pdf"):
        return "\n".join(p.extract_text() or "" for p in PdfReader(path).pages)
    return open(path, encoding="utf-8").read()

ids, docs, metas = [], [], []
for path in glob.glob("notes/**/*", recursive=True):
    if not os.path.isfile(path): continue
    text = load(path)
    for i, chunk in enumerate(chunks(text)):
        ids.append(f"{path}:{i}")
        docs.append(chunk)
        metas.append({"source": path, "chunk": i})

coll.add(ids=ids, documents=docs, metadatas=metas)
print(f"Indexed {len(ids)} chunks.")
    Run it: python ingest.py. Chroma uses a default embedding model, so no extra API setup needed.
  4. Write chat.py — retrieve + generate: 
    import os, chromadb
from anthropic import Anthropic
from dotenv import load_dotenv
load_dotenv()

db = chromadb.PersistentClient(path="./db").get_collection("notes")
llm = Anthropic()

SYSTEM = """You answer questions using ONLY the provided notes.
If the notes don't contain the answer, say so.
Cite sources like [source.pdf:chunk_3] inline."""

def answer(question, k=4):
    hits = db.query(query_texts=[question], n_results=k)
    context = "\n\n".join(
        f"[{m['source']}:{m['chunk']}] {d}"
        for d, m in zip(hits["documents"][0], hits["metadatas"][0])
    )
    msg = llm.messages.create(
        model="claude-haiku-4-5-20251001",
        max_tokens=800,
        system=SYSTEM,
        messages=[{"role": "user",
                   "content": f"Notes:\n{context}\n\nQuestion: {question}"}],
    )
    return msg.content[0].text

if __name__ == "__main__":
    while True:
        q = input("\nAsk: ").strip()
        if not q: break
        print("\n" + answer(q))
    Run it: python chat.py, then ask questions about your notes.
  5. Evaluate honestly. Ask questions your notes should answer, and ones they shouldn't. Does the bot know when to say "I don't know"? If it hallucinates, why?
  6. Tune chunk size, overlap, and k (number of retrieved chunks). Small chunks = precise but lose context. Large chunks = more context but noisier retrieval.
🪄 Common issues… PDFs with poor OCR: pypdf extracts garbage from scanned PDFs. Swap in pytesseract for OCR. Irrelevant retrievals: Your chunks may be too big or your question phrasing too vague. Try hybrid search (keyword + vector). Hallucinations: Tighten the system prompt and add refusal examples.

Try This Next

  • Add a web UI with Gradio or Streamlit. Deploy it on Hugging Face Spaces for free.
  • Measure retrieval quality. Build a test set of 20 Q&A pairs. Compute hit-rate: does the correct source chunk appear in the top-k?
  • Go hybrid. Add BM25 keyword search alongside vector search. Compare the two, then combine.
  • Swap the model. Try GPT-4o-mini, Claude Haiku, and a local Llama model. Compare cost, latency, and answer quality.
  • Add memory. Let the bot remember the last few turns so you can ask follow-ups.
